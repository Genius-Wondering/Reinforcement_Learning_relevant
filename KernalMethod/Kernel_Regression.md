# Kernel Regression（核回归）
---
# 1. Historical Motivation（历史背景）

## 1.1 为什么提出 Kernel Method

1960年代 Nadaraya (1964) 和 Watson (1964) 独立提出核回归，用于非参数密度估计与回归问题。核心动机是：**不预先假设函数的形式**（不像线性回归假设 f 是线性的），而是让数据自己决定函数形状——这是"非参数统计"思想的源头之一。

## 1.2 Linear Basis 的局限

- **Basis需要人工设计**：无论是 Polynomial、Fourier 还是 Spline，都需要提前决定用什么基函数、多少阶——这本质上是在把人对问题的先验强加给模型，选错基函数逼近效果会很差。
- **高维维数爆炸**：多元多项式/样条基的数量随维度指数增长，人工设计在高维下几乎不可行。
- **局部泛化能力差**：全局基函数（如 Polynomial、Fourier）的一个系数变化会影响整个定义域上的预测，即便 RBF/Spline 有一定局部性，中心/节点仍需要预先固定，无法随数据自适应。

## 1.3 Kernel 的核心思想

**为什么不用设计 Basis，而直接利用样本作为 Basis。** 核方法的洞察是：与其预先设计一组固定的基函数 φᵢ(x)，不如让**训练样本本身**充当"基函数的锚点"——每个训练点 xᵢ 通过核函数 K(x,xᵢ) 定义一个以 xᵢ 为中心的局部影响函数，预测值就是所有训练样本影响力的加权平均。这样基函数的数量和位置完全由数据决定，不需要人工设计。

# 2. Mathematical Motivation（数学动机）

## 2.1 从线性模型开始

标准线性模型：

```
f(x) = wᵀφ(x)
```

若 f 与 φ(x) 之间的关系并非线性，需要先做一次**特征映射**（Feature Mapping）φ(x)，把原始输入映射到一个新的空间，使得在新空间里 f 关于映射后的特征是线性的。问题是：φ 应该怎么选？选择不当会导致逼近能力不足或计算代价过高（尤其当需要的 φ 维度很高甚至无穷维时）。

## 2.2 Feature Space

**定义**：φ: X → H，把原始输入空间 X 映射到一个（通常更高维的）特征空间 H。

**为什么映射以后变得线性**：许多在原始空间中非线性可分/非线性关系的问题，在映射到足够高维的空间后会变得线性可分/线性相关——这是核方法背后的几何直觉（类比 SVM 中低维不可分数据映射到高维后线性可分）。

**几何解释**：想象二维平面上呈同心圆分布的两类点，无法用直线分开；但把每个点 (x₁,x₂) 映射到三维 (x₁,x₂,x₁²+x₂²) 后，两类点在第三维上被拉开，可以用一个平面（线性超平面）分开。

## 2.3 Infinite-dimensional Feature Space

**为什么可以无限维**：只要最终的预测公式不需要显式计算 φ(x)，而只需要计算内积 φ(x)ᵀφ(z)，那么 φ 映射到多少维（甚至无穷维）在计算上都不是问题——这正是 2.4 节 Kernel Trick 的核心。

**举 Gaussian Kernel 例子**：高斯核 K(x,z)=exp(−||x−z||²/2σ²) 对应的特征映射 φ(x) 是**无穷维**的（可以证明其等价于把 x 映射到所有阶数的多项式特征的加权组合，权重随阶数增加按阶乘衰减）。这意味着高斯核隐式地在做"无穷阶多项式回归"，但计算代价只有 O(1)（计算一次核函数值），而非无穷。

## 2.4 Kernel Trick

**完整推导**：

给定特征映射 φ，很多学习算法（如线性回归、SVM）的最终求解结果和预测公式，都可以写成只依赖于**内积** φ(xᵢ)ᵀφ(xⱼ) 的形式，而不需要单独知道 φ(xᵢ) 本身。定义：

```
K(x,z) := φ(x)ᵀφ(z)
```

**为什么可以不用知道φ**：只要能设计一个函数 K(x,z) 使其等价于某个（可能未知、可能无穷维）特征映射的内积，就可以直接用 K(x,z) 替换所有出现 φ(x)ᵀφ(z) 的地方，全程不需要显式构造或计算 φ。

**证明 Kernel Trick（存在性方向，Mercer定理简述）**：只要 K(x,z) 满足对称性和半正定性（见第5章），Mercer 定理保证存在某个（可能无穷维的）特征空间 H 和映射 φ: X→H，使得 K(x,z)=⟨φ(x),φ(z)⟩_H 成立。因此，"设计一个核函数"和"设计一个特征映射再取内积"在数学上是等价的，但前者计算代价通常低得多。

# 3. Mathematical Foundations（数学基础）

## 3.1 Hilbert Space

**定义**：Hilbert 空间 H 是一个配备内积 ⟨·,·⟩ 的完备线性空间。

- **Inner Product**：⟨f,g⟩，满足对称性、线性性、正定性（⟨f,f⟩≥0，且=0当且仅当f=0）
- **Norm**：||f|| = √⟨f,f⟩，由内积诱导的范数
- **Projection**：给定 H 的一个闭子空间 V，任意 f∈H 都可以唯一分解为 f = f_V + f_⊥，其中 f_V∈V 是 f 在 V 上的正交投影，f_⊥⊥V。这是最小二乘问题的几何本质——"最优逼近"就是"正交投影"。

## 3.2 Reproducing Kernel Hilbert Space（RKHS）

**为什么叫 RKHS**：因为这个空间中的核函数具有"再生"（reproducing）性质——用核函数本身就可以"再生"出空间中任意函数在任意一点的取值。

**完整定义**：设 H 是定义在 X 上的实值函数构成的 Hilbert 空间。若存在函数 K: X×X→ℝ，使得：
1. 对任意固定 x，K(·,x) ∈ H（核函数关于第一个变量是空间中的一个函数）
2. 对任意 f∈H 和任意 x∈X，有 f(x) = ⟨f, K(·,x)⟩_H（再生性质）

则称 H 是以 K 为核的 RKHS。

**Reproducing Property 逐步证明（存在性构造，Moore-Aronszajn定理简述）**：
- 给定一个对称半正定核 K，先构造预 Hilbert 空间 H₀ = span{K(·,xᵢ) : xᵢ∈X}，即所有形如 Σᵢαᵢ K(·,xᵢ) 的有限线性组合
- 定义内积 ⟨K(·,x), K(·,z)⟩ := K(x,z)（用核函数本身定义内积——这是构造的关键一步）
- 验证该内积良定义（对称、双线性、正定，正定性由 K 的半正定性保证）
- 对 H₀ 做完备化（补上所有 Cauchy 序列的极限）得到 H
- 验证再生性质：⟨f,K(·,x)⟩ = ⟨Σᵢαᵢ K(·,xᵢ), K(·,x)⟩ = Σᵢαᵢ K(xᵢ,x) = f(x) ✓

## 3.3 Mercer Theorem

**为什么 Kernel 一定存在 Feature Space**：Mercer 定理从"分析"的角度证明了任意对称半正定核都可以谱分解为特征函数的加权和，这个分解直接给出了显式（可能无穷维）的特征映射。

**Mercer展开**：设 K 是定义在紧集 X×X 上的连续对称半正定核，则存在正交特征函数 {ψᵢ} 和非负特征值 {λᵢ}，使得：

```
K(x,z) = Σᵢ₌₁^∞ λᵢψᵢ(x)ψᵢ(z)
```

**逐步推导**：
1. 把 K 看作一个积分算子：(T_K f)(x) = ∫K(x,z)f(z)dz
2. K 对称半正定 ⟹ T_K 是自伴、半正定的紧算子
3. 由谱定理，紧自伴算子存在正交特征函数系 {ψᵢ} 和特征值 {λᵢ≥0}，满足 T_K ψᵢ = λᵢψᵢ
4. K 可以用这组特征函数展开为上式（在适当收敛意义下）
5. 令 φ(x) = (√λ₁ψ₁(x), √λ₂ψ₂(x), ...)，则 K(x,z)=φ(x)ᵀφ(z)，φ 即为显式（通常无穷维）特征映射，证明了 K 对应某个特征空间中的内积

## 3.4 Representer Theorem

**为什么最优解一定表示成 f(x)=Σᵢ αᵢ K(x,xᵢ)**：这是核方法最重要的定理，它把一个原本定义在无穷维函数空间 H 上的优化问题，转化为一个只需求解 N 个（N为样本数）实数系数的有限维问题。

**完整证明思路**：设优化问题为：

```
min_{f∈H} L(f(x₁),...,f(xₙ)) + Ω(||f||_H)
```

其中 L 是任意损失函数（只依赖于 f 在训练点上的取值），Ω 是关于范数的单调递增正则化函数。

- 将 H 分解为两部分：H = span{K(·,xᵢ)}ᵢ₌₁ᴺ ⊕ H⊥（正交补空间）
- 任意 f∈H 可写成 f = Σᵢαᵢ K(·,xᵢ) + f⊥，f⊥ ∈ H⊥
- 由再生性质，f(xⱼ) = ⟨f,K(·,xⱼ)⟩ = Σᵢαᵢ K(xᵢ,xⱼ) + ⟨f⊥,K(·,xⱼ)⟩ = Σᵢαᵢ K(xᵢ,xⱼ)（因为 K(·,xⱼ)∈span{K(·,xᵢ)}，与 H⊥ 正交，第二项恒为0）
- 即 **f⊥ 完全不影响损失函数 L 的取值**（因为 L 只依赖训练点上的函数值，而这些值与 f⊥ 无关）
- 但 ||f||² = ||Σαᵢ K(·,xᵢ)||² + ||f⊥||²（勾股定理，正交分解），f⊥≠0 只会增大正则化项 Ω(||f||)
- 因此最优解一定有 f⊥=0，即最优 f 必然形如 f(x)=Σᵢαᵢ K(x,xᵢ)，证毕。

# 4. Mathematical Definition（数学定义）

统一数学形式：

```
f̂(x) = Σᵢ₌₁ᴺ αᵢ K(x,xᵢ)
```

Kernel Regression（Nadaraya-Watson 估计）的具体定义：

```
f̂(x) = Σᵢ K(x,xᵢ)yᵢ / Σⱼ K(x,xⱼ)
```

**全部符号解释**：
- xᵢ, yᵢ：第 i 个训练样本的输入与输出
- K(x,z)：核函数，衡量 x 与 z 的"相似度"
- αᵢ：核回归中直接等于（归一化后的）权重，等价于局部加权平均
- **Kernel Matrix / Gram Matrix**：K_{ij} = K(xᵢ,xⱼ)，N×N 矩阵，记录所有训练样本两两之间的核函数值，是核方法计算的核心对象

# 5. Kernel Function（核函数）

**为什么叫 Kernel**：源自积分算子理论中"核"（积分核）的概念——K(x,z) 是定义在两个变量上、决定一个积分变换性质的函数。

**核函数需要满足什么性质**：
- **Symmetric（对称性）**：K(x,z) = K(z,x)
- **Positive Semi-definite（半正定性）**：对任意有限点集 {x₁,...,xₙ} 和任意实数 {c₁,...,cₙ}，有 Σᵢⱼ cᵢcⱼK(xᵢ,xⱼ) ≥ 0

**证明（半正定性与内积形式的等价性）**：若 K(x,z)=φ(x)ᵀφ(z)，则 ΣᵢⱼcᵢcⱼK(xᵢ,xⱼ) = Σᵢⱼcᵢcⱼφ(xᵢ)ᵀφ(xⱼ) = ||Σᵢcᵢφ(xᵢ)||² ≥ 0，天然满足半正定性；反过来，Mercer定理保证半正定核必然对应某个内积形式（3.3节已证明）。

## 常见 Kernel

**Linear**：K(x,z)=xᵀz。几何意义：退化为普通线性模型。适用场景：数据本身接近线性关系，或作为基线对比。

**Polynomial**：K(x,z)=(xᵀz+c)^d。几何意义：隐式对应所有阶数≤d的多项式特征交叉项。适用场景：特征间存在有限阶多项式交互效应。

**Gaussian（RBF Kernel）**：K(x,z)=exp(−||x−z||²/2σ²)。几何意义：以距离衰减的相似度，对应无穷维特征空间。适用场景：默认首选，适合大多数光滑函数逼近任务。

**Laplacian**：K(x,z)=exp(−||x−z||/σ)。几何意义：类似高斯但衰减更慢（尖峰状），对异常值/非光滑结构更鲁棒。适用场景：数据存在局部尖锐变化的场景。

**Sigmoid**：K(x,z)=tanh(κxᵀz+c)。几何意义：模拟神经网络单隐层的激活函数，但不一定满足半正定性（并非严格Mercer核）。适用场景：历史上用于模拟神经网络行为，现代已较少使用。

**Matérn**：K(x,z)=(基于贝塞尔函数的距离衰减核，含光滑度参数ν)。几何意义：ν→∞时退化为高斯核，ν较小时对应更"粗糙"（less smooth）的函数先验。适用场景：高斯过程中控制函数光滑度的标准选择，机器人/地理空间建模常用。

# 6. Approximation Formula（逼近公式）

统一写成：

```
f(x) = Σᵢ αᵢ K(x,xᵢ)
```

**逐步推导**：由第3.4节 Representer Theorem，任何在 RKHS 中、损失函数只依赖训练点取值的优化问题，其最优解都必然落在 span{K(·,xᵢ)} 中，因此可以写成训练样本核函数的线性组合。

**为什么样本就是 Basis**：对比 Linear Basis Expansion（第一类方法）中人工设计的 φᵢ(x)，这里的"基函数"变成了 K(x,xᵢ)——每一个训练样本 xᵢ 都提供一个基函数 K(·,xᵢ)，基函数的数量和位置完全由数据决定，不需要人工指定。

**为什么属于 Non-parametric**：模型的复杂度（有效参数数量 N）随训练样本数量增长而增长，不是固定的（区别于线性基展开中基函数数量是预先固定的超参数）——这是"非参数"方法的定义性特征。

# 7. Optimization（优化）

**如何学习 α**：

**Least Squares（核回归/核最小二乘）**：最小化 Σᵢ(yᵢ−f̂(xᵢ))²，若不加正则化，直接令 f̂(xᵢ)=yᵢ（插值解）可能导致 K 不可逆或过拟合，因此实践中几乎总是加正则化。

**Regularization / Kernel Ridge Regression**：

```
min_α ||y−Kα||² + λαᵀKα
```

对 α 求导并令梯度为0：

```
−2K(y−Kα) + 2λKα = 0
K(y−Kα) = λKα
K(y − Kα − λα) = 0
```

（假设 K 可逆，或在 K 的值域内）可得：

```
Ky = K²α + λKα = K(K+λI)α
y = (K+λI)α
α = (K+λI)⁻¹y
```

**Dual Problem / Lagrange（以 SVR 为例的一般套路）**：对于带约束的核方法（如 SVR），原问题（primal）在无穷维/高维特征空间中不便直接求解，通过引入 Lagrange 乘子将其转化为只涉及内积（即核函数）的对偶问题（dual），对偶问题的变量数等于样本数 N，与特征空间维度无关——这是核方法能处理无穷维特征空间的计算基础。

# 8. Approximation Theory（逼近理论）

**为什么 Kernel 可以逼近任意连续函数**：若核函数是**Universal Kernel**（如高斯核），其对应的 RKHS 在紧集上关于一致范数稠密于连续函数空间 C(X)——即对任意连续函数和任意精度 ε，存在该 RKHS 中的函数使得两者的最大误差小于 ε。

**Universal Kernel**：Steinwart (2001) 给出了核函数具有普适逼近性的充要条件（如平移不变核的傅里叶变换处处为正）；高斯核满足该条件，是 Universal Kernel 的代表。

**Consistency（一致性）**：随着样本量 N→∞，核回归（Nadaraya-Watson估计量）在带宽选择合理（如带宽随N按特定速率趋于0）的条件下，依概率收敛到真实函数 f。

**Approximation Error / Generalization Error 分解**：与第零章的误差分解一致——核方法的总误差 = RKHS 空间的逼近误差（若核是Universal Kernel则该项可忽略）+ 有限样本下的估计误差（依赖 N 和正则化强度 λ）。

**Bias-Variance**：正则化系数 λ（或带宽 σ）越大，模型越平滑，Bias 增大、Variance 减小；λ越小，模型越贴合训练数据，Bias减小、Variance增大——这是核方法中最核心的超参数权衡。

# 9. Computational Complexity（计算复杂度）

- **Kernel Matrix**：N×N，存储复杂度 O(N²)
- **训练复杂度**：Kernel Ridge 需要对 (K+λI) 求逆，复杂度 O(N³)
- **预测复杂度**：单点预测需要计算该点与所有训练点的核函数值，O(N)（若训练集很大，代价可观）
- **内存复杂度**：O(N²)（存储 Gram 矩阵）

**为什么 O(N²)**：因为核方法本质上要计算所有训练样本两两之间的核函数值（Gram矩阵），样本对的数量是 N²。

**为什么 GP 是 O(N³)**：高斯过程回归需要对协方差矩阵（即Gram矩阵）求逆或做 Cholesky 分解以计算后验均值和方差，矩阵求逆/分解的标准复杂度是 O(N³)。

**如何降低复杂度**：
- **Nyström 方法**：用 M≪N 个"地标点"（landmark points）低秩近似整个 Gram 矩阵，将复杂度降到 O(NM²)
- **Random Features（随机特征，如 Random Fourier Features）**：用有限维随机特征近似核函数的隐式无穷维映射，将核方法转化为显式线性回归，训练复杂度降为 O(ND²)（D为随机特征维度）
- **Sparse GP（稀疏高斯过程）**：用少量诱导点（inducing points）近似完整GP后验，复杂度从 O(N³) 降到 O(NM²)（M为诱导点数）

# 10. RL中的应用

**Kernel TD**：将价值函数表示为 V̂(s) = Σᵢαᵢ K(s,sᵢ)，TD误差 δ = r + γV̂(s') − V̂(s)，用核化的半梯度更新 α（本质是在RKHS中做函数逼近版的TD学习）。

**Kernel SARSA**：类似地将 Q̂(s,a) = Σᵢαᵢ K((s,a),(sᵢ,aᵢ)) 核化，用于连续状态-动作空间的on-policy控制。

**Kernel LSPI（Least-Squares Policy Iteration）**：Xu et al. (2007) 将 LSPI 核化，用核函数替代线性基函数，在贝尔曼方程的最小二乘求解中引入 Representer Theorem，使得最优解自动落在训练样本张成的核函数空间中。

**Gaussian Process RL**：用GP对价值函数（或Q函数、动力学模型）做贝叶斯推断，天然提供不确定性估计，用于指导探索（如GP-UCB类算法）。

**Kernel Bellman Equation（数学推导）**：贝尔曼方程 V=R+γPV 在核化设定下写作：

```
V̂(s) = Σᵢαᵢ K(s,sᵢ)，需求解 α 使得投影贝尔曼残差最小
```

即 min_α ||ΠK(R+γPV̂) − V̂||²，其中 Π 是到 span{K(·,sᵢ)} 的投影算子——这与线性函数逼近下的投影贝尔曼方程（第六章理论分析中的 MSPBE）形式一致，只是把线性基换成了核函数基。

# 11. Convergence Theory（收敛理论）

**Kernel TD 是否收敛**：与线性TD类似，Kernel TD 本质上仍是"半梯度 + 线性于（核）特征"的方法（核函数值本身可视为高维/无穷维特征），因此在 on-policy 设定下满足与线性TD相似的收敛性条件（Tsitsiklis & Van Roy理论的核化推广）；off-policy 设定下同样面临 Deadly Triad 问题，需要类似 Gradient-TD 的核化改进。

**Kernel Bellman Projection**：与线性情形一样，核化TD实际上是在求解投影到 span{K(·,sᵢ)} 子空间上的近似贝尔曼不动点，而非真实贝尔曼方程的精确解。

**Projected Fixed Point**：只要投影算子 Π（到核函数张成子空间的投影）和贝尔曼算子的复合映射仍是压缩映射（在适当范数下），则核化TD的不动点存在且唯一。

**误差界**：核化方法的总误差可分解为（RKHS空间对真实价值函数的）逼近误差 + （有限样本下的）估计误差 + TD算法本身的（半梯度导致的）投影误差，三者叠加给出最终的价值函数逼近误差上界。

---
# 12. Advantages

- 无需人工设计基函数，模型复杂度随数据自适应增长
- 理论完备（RKHS、Representer Theorem、Mercer定理构成严密的数学体系）
- 通过 Universal Kernel 可保证对任意连续函数的逼近能力
- 天然支持贝叶斯扩展（Gaussian Process），提供不确定性估计

---
# 13. Disadvantages

- 计算复杂度高（O(N²)~O(N³)），难以直接扩展到大规模数据集
- 核函数及其超参数（带宽、正则化系数）的选择仍需要调参或交叉验证
- 预测阶段需要保留全部（或大部分）训练样本，存储和推理成本随数据量线性增长

---
# 14. Comparison

| 方法 | 基函数来源 | 参数量 | 高维扩展性 | 计算复杂度 |
|---|---|---|---|---|
| Kernel Methods | 数据驱动（每个样本一个基） | 随N增长（非参数） | 好（核trick避开维度灾难） | 高 O(N²)~O(N³) |
| Linear Basis | 人工设计 | 固定 | 差（维度灾难） | 低 |
| Tree | 数据驱动（空间递归划分） | 随树规模增长 | 中 | 中 O(N log N) |
| Neural Network (MLP) | 数据驱动（端到端学习） | 固定（网络结构） | 好 | 训练高，预测低 |

---
# 15. Applications

- **机器人**：连续状态空间的价值函数/动力学模型逼近（机械臂控制、步态学习）
- **控制**：核化LQR、核化MPC中的非线性系统建模
- **Bayesian RL**：利用GP的后验不确定性指导探索-利用权衡
- **GP-RL**：PILCO等基于高斯过程模型的数据高效强化学习框架，在样本极其昂贵的真实机器人任务中表现突出

# Decision Tree（决策树）
---
# 1. Historical Motivation（历史背景）

**为什么提出 Tree**：决策树的思想可追溯到统计学中的自动交互检测（AID, Morgan & Sonquist, 1963），1980年代 Breiman 等人（CART, 1984）与 Quinlan（ID3, 1986; C4.5, 1993）将其系统化为现代机器学习方法。核心动机是构造一种**可解释、无需假设函数光滑形式、能自动发现特征交互**的非线性逼近器。

**为什么不用线性模型**：线性模型（及Linear Basis Expansion方法）假设函数关于（变换后的）输入是线性的，且需要人工设计特征交互项；当真实函数存在复杂的、非光滑的、依赖多特征组合的阈值型规律时（例如"如果年龄>30且收入>5万则..."），线性模型难以自然表达这类逻辑，需要人工构造大量交叉特征。

**为什么不用 Kernel**：核方法虽然能捕捉复杂非线性关系，但（1）计算复杂度高（O(N²)~O(N³)），（2）预测结果不易解释（核函数的加权和缺乏直观意义），（3）核方法的"相似度"是全局连续的距离度量，难以自然表达"离散规则/阈值判断"这种结构。

**Tree 能解决什么问题**：决策树通过递归地对输入空间做**轴对齐的矩形划分**，在每个划分区域内用简单常数（或简单模型）逼近函数——既不需要预先设计基函数，又天然可解释（每条从根到叶的路径对应一条可读的if-then规则），且计算速度快（预测只需沿树走一条路径，O(depth)）。

# 2. Mathematical Motivation（数学动机）

这一章重点解释：**为什么可以把函数逼近转化成空间划分。**

从 f(x) 到 Partition 到 Piecewise Constant 的逐步推导：

1. **出发点**：任意函数 f(x) 在其定义域 X 上取值可能千变万化，但若将 X 划分成足够小的子区域 R₁,...,R_M，则在每个足够小的子区域内，f 的取值变化很小，可以近似看作常数（这本质上是黎曼积分/分段常数逼近的思想，与第一类方法中 Tile Coding 的数学基础完全一致）。

2. **划分（Partition）**：X = R₁ ∪ R₂ ∪ ... ∪ R_M，且 Rᵢ ∩ Rⱼ = ∅（i≠j），即区域两两不重叠、完全覆盖定义域。

3. **分段常数逼近（Piecewise Constant）**：f̂(x) = Σᵢ cᵢ·I(x∈Rᵢ)，其中 cᵢ 是区域 Rᵢ 内的代表值（通常取该区域内训练样本目标值的均值，这是使均方误差最小的最优常数选择）。

4. **决策树的独特之处**：与 Tile Coding 的预先固定网格不同，决策树**用数据驱动的方式自适应地决定划分方式**——在哪个特征、哪个阈值上切一刀，是通过优化划分准则（第5章）从数据中学出来的，而不是人工预先设定的均匀网格。这正是"树"与"网格"最本质的区别：网格是各维度独立均匀切分，树是**递归、自适应、可以在不同区域用不同分辨率切分**的划分方式。

# 3. Mathematical Foundations（数学基础）

这是整个 Tree 最重要部分。

## 3.1 Recursive Space Partition

**为什么不断切空间**：单次划分（把整个空间切成2块）通常表达能力不足；决策树通过**递归地**对已划分出的子区域继续划分（每次都是在当前子区域内选一个特征和阈值切一刀），使得整体划分可以任意精细、任意不规则（在轴对齐的限制下）地逼近复杂的函数结构。

**数学定义**：树的每个内部节点对应一次二元划分决策：

```
node: 若 x_j ≤ t，进入左子树；否则进入右子树
```

其中 j 是选定的特征下标，t 是选定的阈值。根节点对应整个输入空间 X，每次划分把当前节点对应的区域 R 分裂为：

```
R_left = {x∈R : x_j≤t}，R_right = {x∈R : x_j>t}
```

**超矩形（Hyper-rectangle）**：由于每次划分都是"某一维 ≤ 阈值"的形式（轴对齐），任意叶节点对应的区域 Rᵢ 都是输入空间中的一个超矩形（各维度独立的区间乘积 R_i = ∏_d [l_d, u_d]）。

**Region**：叶节点对应的最终划分区域，是整个决策树函数逼近的"基本单元"，等价于第一类方法中的"局部基函数支撑区域"，但形状由数据驱动而非预先设定。

## 3.2 Piecewise Constant Approximation

**证明 Tree 本质就是 f(x)=Σᵢ cᵢI(x∈Rᵢ)**：

- 决策树的预测过程：给定输入 x，从根节点出发，根据每个内部节点的划分规则走到某个唯一的叶节点 Rᵢ
- 该叶节点对应一个固定的预测值 cᵢ（回归树中通常是该叶节点内训练样本目标值的均值）
- 因此对任意 x，f̂(x) = cᵢ，其中 i 是 x 所属的叶节点编号
- 这可以等价写成 f̂(x) = Σᵢ cᵢ·I(x∈Rᵢ)，因为 I(x∈Rᵢ) 对唯一的正确叶节点为1，对其余叶节点为0（叶节点区域两两不相交且完全覆盖空间），求和自动"选出"正确的 cᵢ
- 这与 Tile Coding 的逼近公式 f(x)=Σwᵢφᵢ(x)（φᵢ为指示函数）在数学形式上完全一致，唯一区别是区域 Rᵢ 的形状/位置由数据驱动的递归划分决定，而非预先固定的均匀网格

## 3.3 Piecewise Linear Approximation

**为什么有些 Tree 可以线性**：标准CART在叶节点用常数逼近（Piecewise Constant），但可以将叶节点的预测模型从"常数"替换为"局部线性回归"（Model Tree，如 M5 算法），即 f̂(x)=Σᵢ(wᵢᵀx+bᵢ)·I(x∈Rᵢ)。

**什么时候变成 Piecewise Linear**：当目标函数在局部区域内接近线性、但全局非线性/非光滑时（如分段线性函数、或整体趋势线性但存在若干结构性断点的函数），Model Tree 用更少的划分就能达到与纯常数树相同的精度——本质上是"用局部线性模型替代局部常数模型"，划分标准和树结构生长逻辑仍遵循3.1节的递归划分框架。

## 3.4 Empirical Risk Minimization

**Tree 为什么选择最优划分**：决策树在每个节点选择使**训练误差下降最多**的（特征，阈值）组合进行划分，这是经验风险最小化（ERM）原则在树结构上的贪心应用。

**ERM 推导**：给定节点 R 及其内部样本，选择划分 (j,t) 使得划分后左右子区域的加权误差之和最小：

```
(j*, t*) = argmin_{j,t} [ |R_left|/|R| · L(R_left) + |R_right|/|R| · L(R_right) ]
```

其中 L(·) 是节点内的损失（回归任务中通常是方差/均方误差，见第5章），|R| 表示区域内样本数量。这等价于最大化划分带来的**损失下降量** ΔL = L(R) − [|R_left|/|R|·L(R_left) + |R_right|/|R|·L(R_right)]。

# 4. Mathematical Definition（数学定义）

- **Node（节点）**：树中的一个决策单元，内部节点包含划分规则 (j,t)，叶节点包含预测值 c
- **Leaf（叶节点）**：不再继续划分的终端节点，对应一个区域 Rᵢ 和预测值 cᵢ
- **Split（划分）**：一次二元决策 x_j≤t，将当前区域分裂为左右两个子区域
- **Depth（深度）**：从根节点到某节点经过的划分次数，控制模型复杂度
- **Region（区域）**：某节点（尤其叶节点）对应的输入空间子集
- **Impurity（不纯度）**：衡量区域内目标值的"混乱程度"（回归中常用方差，分类中常用熵/Gini），是划分准则的核心度量

全部数学符号说明：j（特征下标）、t（阈值）、Rᵢ（第i个区域/叶节点）、cᵢ（叶节点预测值）、N_R（区域R内样本数）、y_R（区域R内样本目标值集合）。

# 5. Split Criterion（划分准则）

这是最重要章节。每一种全部推导。

**Regression / Variance Reduction（回归/方差降低准则）**：

节点 R 的不纯度定义为方差：

```
L(R) = (1/N_R) Σ_{i∈R} (yᵢ − ȳ_R)²，ȳ_R = (1/N_R)Σ_{i∈R}yᵢ
```

划分准则：选择 (j,t) 最大化方差降低量：

```
ΔVar = L(R) − [N_left/N_R · L(R_left) + N_right/N_R · L(R_right)]
```

**推导为什么均值是最优常数**：在区域R内用常数c预测，最小化 Σ(yᵢ−c)² 对c求导为零，得 c*=ȳ_R（均值），代入得到 L(R) 恰好是该区域用最优常数预测时的最小均方误差——这也是为什么"方差"和"均方误差"在决策树语境下可以互换使用。

**Classification / Entropy / Information Gain（分类/熵/信息增益）**：

节点不纯度定义为熵：

```
H(R) = −Σ_k p_k log(p_k)，p_k = 区域R内类别k样本比例
```

信息增益：

```
IG = H(R) − [N_left/N_R·H(R_left) + N_right/N_R·H(R_right)]
```

**推导**：熵衡量类别分布的不确定性，H(R)=0当且仅当区域内样本全为同一类别（纯度最高）；划分后子区域熵的加权平均越小，说明划分越"有效地"分离了不同类别，信息增益即为划分前后熵的下降量，与互信息 I(Y;划分变量) 在数学上等价。

**Gini 不纯度**：

```
Gini(R) = 1 − Σ_k p_k² = Σ_k p_k(1−p_k)
```

**推导**：Gini指数等价于"随机抽取两个样本，其类别不同的概率"——Σ_{k≠k'} p_k p_{k'} = Σ_k p_k(1−p_k)，是熵的一种计算更快（无需对数运算）的近似替代，CART算法默认使用Gini准则。

# 6. Optimization（训练）

**Tree 如何长出来**：从根节点开始，在整个训练集上遍历所有特征、所有可能阈值，选择使损失下降最多的划分；对划分后的左右子节点递归重复该过程，直到满足停止条件（如达到最大深度、节点样本数过少、或不纯度下降量小于阈值）。

**Greedy（贪心）**：每一步划分只考虑当前节点的局部最优，不考虑该划分对后续所有子树整体结构的全局影响。

**为什么 NP-Hard**：Hyafil & Rivest (1976) 证明了构造使总体误差最小的**最优二叉决策树**是NP-hard问题——因为每个划分点的选择会影响所有后续子树可能达到的最优结构，穷举所有可能的树结构组合数随特征数和样本数呈指数增长（组合优化的经典困难性）。

**为什么只能 Greedy**：由于全局最优不可计算，实践中普遍采用贪心策略——每步局部最优地选择当前最好的划分，虽不保证全局最优，但计算效率高（多项式时间）且在实践中效果良好。

**剪枝（Pruning）**：贪心生长容易导致树过深、过拟合（每个叶节点样本极少，方差主导误差），需要剪枝控制复杂度。

**Cost Complexity（代价复杂度剪枝，CART标准方法）**：

定义带复杂度惩罚的目标：

```
C_α(T) = Σ_{leaf∈T} L(leaf) + α|T|
```

其中 |T| 是叶节点数量，α≥0 是复杂度惩罚系数。**推导**：对固定的 α，存在唯一的最小化 C_α(T) 的子树 T_α（可以证明这些最优子树随 α 增大形成一个嵌套序列）；通过交叉验证在这个嵌套序列中选择最优的 α（进而确定最优树规模），这是"先长满整棵树再剪枝"相比"提前停止生长（pre-pruning）"更稳健的原因——提前停止可能因为某次划分局部收益不明显而错过后续更有价值的划分。

# 7. Approximation Theory（逼近能力）

**为什么 Piecewise Constant 可以逼近任意函数**：与Tile Coding（第一类方法第4篇）的逼近理论完全同源——分段常数函数集合，当划分区域的最大直径趋于0（即树的深度趋于无穷、每个区域收缩为一点）时，可以在L²或一致范数意义下逼近任意有界（分段）连续函数（黎曼和思想的推广）。

**一致收敛**：若目标函数f一致连续，且树的划分能保证每个叶节点区域的直径随树深度增加而趋于0，则决策树逼近在整个定义域上一致收敛到f。

**逼近误差**：主要由叶节点区域内部f的变化幅度（即该区域内f的振荡范围）决定，与区域直径大致成正比（一阶收敛，与Tile Coding一致，收敛阶数低于光滑基函数方法）。

**Bias**：树深度越浅（叶节点越少），每个区域越大，区域内用单一常数逼近的偏差越大——高Bias。

**Variance**：树深度越深（叶节点越多），每个区域样本数越少，叶节点的均值估计方差越大（对训练数据的随机扰动敏感）——高Variance。深度是决策树中控制Bias-Variance权衡的核心超参数。

**VC Dimension**：深度为d的二叉决策树，其表达能力（可实现的划分方式数量）与叶节点数 2^d 及特征维度相关；一般地，决策树类的VC维随树的复杂度（节点数）近似线性增长，这为其泛化误差界提供了统计学习理论工具（决策树是历史上VC理论最早系统应用的非参数方法之一）。

# 8. Ensemble Theory（集成理论）

**Extra Tree（Extremely Randomized Trees）**：在标准决策树基础上，划分时不仅随机选择候选特征子集，**阈值 t 也随机选取**（而非遍历寻找最优阈值），进一步降低树与树之间的相关性。推导：假设单棵树的方差为 σ²，树间平均相关系数为 ρ，则 M 棵树平均后的方差为 ρσ² + (1−ρ)σ²/M；Extra Tree 通过随机化阈值进一步降低 ρ，使集成方差降得更低（代价是单棵树的Bias略微增加）。

**Random Forest**：Breiman (2001) 提出，结合两种随机化：（1）Bagging——对训练集做有放回重采样（bootstrap），每棵树用不同的重采样数据训练；（2）特征子采样——每次划分只从随机选取的特征子集中寻找最优划分。

**为什么 Bagging 降低方差（完整数学解释）**：设 M 棵树的预测为 f̂₁,...,f̂_M，若它们是独立同分布、方差均为σ²，则平均预测 f̄=(1/M)Σf̂ₘ 的方差为 σ²/M（方差随树数量线性减小）；但实际中各树因使用同一训练集的重采样版本而**正相关**（相关系数ρ>0），此时：

```
Var(f̄) = ρσ² + (1−ρ)σ²/M
```

当 M→∞，方差收敛到 ρσ²（而非0），因此**降低树间相关性ρ**（通过随机特征子采样等手段）比单纯增加树的数量M更有效地降低集成方差——这正是Random Forest在Bagging基础上额外引入特征随机化的数学动机。

**Gradient Boosting**：与Bagging（并行、独立训练多棵树取平均）不同，Boosting是**串行**训练：每一步训练一棵新树去拟合当前模型的残差（或更一般地，负梯度方向）。

**为什么 Boosting 降低 Bias（完整数学解释）**：设当前模型为 F_{m-1}(x)，目标是最小化损失 L(y,F(x))。Gradient Boosting在第m步构造：

```
F_m(x) = F_{m-1}(x) + η·h_m(x)
```

其中 h_m(x) 是拟合负梯度 −∂L(y,F_{m-1}(x))/∂F_{m-1}(x) 的回归树（对平方损失，负梯度恰好等于残差 y−F_{m-1}(x)），η是学习率。**推导**：这是函数空间中的梯度下降——把每棵树 h_m 看作"搜索方向"，通过不断沿负梯度方向添加新树，逐步降低训练损失，即降低Bias（起始模型F₀通常是极简单的常数预测，Bias很高，随着树的添加不断修正逼近能力不足的部分）；由于是串行拟合残差而非并行平均独立模型，Boosting主要作用是降低Bias而非降低Variance（因此Boosting对过拟合更敏感，通常需要额外的正则化，如学习率收缩、树的深度限制、子采样等）。

# 9. Computational Complexity（计算复杂度）

**建树复杂度**：对每个节点寻找最优划分需要遍历所有特征（d个）和该节点内所有样本可能的阈值（排序后为O(N)个候选），单节点复杂度O(dN log N)（含排序）；完整建树（假设树平衡，深度O(log N)）总复杂度约为 **O(dN log²N)**。

**预测复杂度**：单样本预测只需沿树走一条路径，复杂度为 **O(depth) = O(log N)**（平衡树情形），是所有方法中预测速度最快的一类之一。

**内存复杂度**：O(节点数)，通常与训练样本数同阶或更少（取决于剪枝程度）。

**并行复杂度**：单棵决策树的构建过程本质上是串行的（子节点的划分依赖父节点的划分结果），但**不同分支之间可以并行**；集成方法中，Random Forest的多棵树相互独立，可以完全并行训练（这是Random Forest相比Gradient Boosting在工程上的一大优势——Boosting的串行依赖结构天然难以并行化，虽然现代实现如XGBoost/LightGBM在单棵树内部的划分搜索上做了大量并行优化）。

# 10. RL中的应用

**Fitted Q Iteration（FQI，数学公式）**：将Q函数逼近为回归树（或树集成），通过迭代方式求解贝尔曼最优方程：

```
Q_{k+1} = argmin_Q Σᵢ (Q(sᵢ,aᵢ) − [rᵢ + γ·max_a' Q_k(sᵢ',a')])²
```

每一轮迭代：用当前Q_k计算所有样本的TD目标 yᵢ=rᵢ+γmax_a'Q_k(sᵢ',a')，然后训练一棵新的回归树（或树集成）拟合 {(sᵢ,aᵢ), yᵢ} 得到 Q_{k+1}，反复迭代直至收敛——这是离线（batch）RL中最经典的树方法应用（Ernst et al. 2005）。

**Extra Tree RL**：Ernst et al. 发现用Extra Trees作为FQI中的回归器，比标准CART或核方法在多个连续控制基准任务上表现更稳定、样本效率更高，成为FQI的事实标准实现。

**Tree Backup（树回溯算法，注意与"回归树"是完全不同的概念）**：一种off-policy的多步TD算法（Precup et al. 2000），用"期望下的树状展开"处理off-policy修正（避免重要性采样的高方差），更新公式：

```
δₜ = rₜ₊₁ + γΣ_a π(a|sₜ₊₁)Q(sₜ₊₁,a) − Q(sₜ,aₜ)
```

该算法名称中的"树"指的是对未来所有可能动作分支做期望展开（类似决策树的分支结构），而非使用回归树作为函数逼近器，是RL文献中容易混淆的两个不同"树"概念。

**Decision Tree Policy（决策树策略）**：直接用决策树参数化策略π(a|s)，每个叶节点输出一个动作（或动作分布），优势是可解释性强（策略可以直接读作if-then规则），常用于需要策略透明度的安全关键场景（如医疗、金融决策）。

**Model Tree（用于Model-based RL）**：用Model Tree（叶节点为局部线性模型而非常数）逼近环境的状态转移函数或奖励函数，比常数树能更精确地捕捉局部动力学的线性趋势，同时保留树结构对非线性/分段结构的表达能力。

# 11. Convergence Theory

**Tree FQI（拟合Q迭代的收敛性）**：与线性/核化情形类似，FQI的每一轮迭代都可以看作贝尔曼最优算子 T* 与回归算子（树拟合，可视为到"树可表示函数集合"上的某种非线性投影 Π_tree）的复合：

```
Q_{k+1} = Π_tree(T* Q_k)
```

**Error Propagation（误差传播）**：设每一轮回归拟合的误差为 εₖ = ||Π_tree(T*Q_k) − T*Q_k||_∞（树逼近能力有限导致的拟合误差），可以证明经过K轮迭代后，最终价值函数误差被限制在：

```
||Q_K − Q*||_∞ ≤ (2γ/(1−γ)²) · max_k εₖ + O(γ^K)
```

（这是Munos & Szepesvári 2008等FQI误差传播理论的标准形式）——即使每一步的树拟合误差εₖ有界，只要贝尔曼算子的压缩系数γ<1，累积误差仍然有界，但误差会被 1/(1−γ)² 放大，说明γ接近1时（长期规划问题）FQI对每一步回归误差非常敏感。

**Bellman Backup**：与线性/核方法一致，FQI本质上是在有限表达能力的函数类（树/树集成可表示的分段常数或分段线性函数）上求解投影贝尔曼不动点的近似算法，不保证收敛到精确的Q*，只保证收敛到某个由εₖ控制的邻域内。

**Approximation Error / Estimation Error**：Approximation Error由树/树集成的表达能力决定（叶节点数量、深度上限）；Estimation Error由每轮迭代使用的样本数量决定（样本越多，每个叶节点内的均值估计越准确）——与第零章的误差分解框架完全一致。

# 12. Advantages

- 可解释性强，预测路径可直接读作if-then规则
- 无需对输入做归一化/标准化，天然处理混合类型特征（数值+类别）
- 能自动捕捉特征间的非线性交互，无需人工设计交叉特征
- 预测速度快（O(log N)），适合实时/低延迟场景
- 对异常值相对鲁棒（划分基于阈值排序而非距离，不受极端值的数值大小影响）

---
# 13. Disadvantages

- 单棵树容易过拟合（贪心生长+高方差），需要剪枝或集成方法缓解
- 轴对齐划分对某些函数结构（如对角线性边界）表达效率低，需要很多次划分才能逼近
- 分段常数逼近本身收敛阶数低（一阶），不适合高精度光滑函数拟合
- 贪心生长不保证全局最优，且对训练数据的微小扰动可能导致树结构剧烈变化（不稳定性）

---
# 14. Comparison

| 方法 | 可解释性 | 局部性 | 平滑度 | 高维扩展性 | 训练/预测速度 |
|---|---|---|---|---|---|
| Decision Tree | 很好 | 好（区域局部） | 差（不连续） | 好 | 训练快，预测很快 |
| Linear Basis | 中 | 依方法而定 | 好 | 差 | 快 |
| Kernel | 差 | 好（依带宽） | 好 | 中（核trick） | 训练慢O(N²)~O(N³) |
| MLP | 差 | 无天然局部性 | 好 | 好 | 训练中等，预测快 |
| Gradient Boosting | 中（集成后降低） | 好 | 中（分段，但更精细） | 好 | 训练中等，预测快 |

# 15. Applications

- **Control**：FQI及其树集成变体在经典连续控制基准（如倒立摆、Puddle World）中的离线策略学习
- **Offline RL**：树方法因其对分布外样本相对保守（不会像神经网络那样在未见过区域产生剧烈外推），是早期离线RL的主流函数逼近器
- **Robot**：机器人离线批量学习场景（结合真实交互数据有限、需要样本高效算法的特点）
- **Medical**：医疗决策支持系统中，决策树策略因高可解释性被用于给出可审查的治疗建议
- **Finance**：信用评分、风控决策树，Gradient Boosting（如XGBoost）在金融风控建模中的广泛工业应用

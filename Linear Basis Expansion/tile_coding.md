# Tile Coding（瓦片编码）
---
# 1. Motivation（为什么提出）

**历史背景**：Tile Coding 是 CMAC（Albus, 1975）思想的简化和推广，在 Sutton 的早期RL工作中被大量使用，是最早被系统用于强化学习函数逼近的方法之一。

**解决什么问题**：需要一种**计算极快**（每次只需查表，不用做浮点乘法求和）、**局部性好**（更新只影响局部区域，不影响全局）的粗编码（coarse coding）方法，用于在线、实时的RL更新。

**为什么以前的方法不好**：Polynomial/Fourier 等全局基每次预测和更新都需要计算所有基函数的值（计算量随基函数总数线性增长）；Tile Coding 利用离散化 + 稀疏激活的特点，把大部分计算量降到 O(1)（只需查找当前激活的少数几个瓦片）。

---
# 2. Mathematical Idea（数学思想）

**一句话解释**：把连续状态空间切成多套**互相偏移**的网格（瓦片），每套网格对状态做粗略离散化，多套叠加实现精细的分辨率。

**几何解释**：单独一套网格（tiling）分辨率很粗，但多套网格以不同偏移量叠加后，任意两个相近但不同的状态点落入的瓦片组合几乎不完全相同，从而恢复出比单套网格精细得多的有效分辨率。

**数学解释**：这是一种分段常数函数的叠加逼近，每个瓦片对应一个指示函数（indicator function），状态激活的瓦片集合构成一个稀疏二值特征向量。

---
# 3. Mathematical Definition（数学定义）

给定 m 套 tiling（每套网格相对彼此有不同偏移），每套 tiling 把状态空间分割成若干瓦片。定义指示基函数：

```
φᵢ(x) = 1[x 落在瓦片 i 内]，否则为 0
```

逼近为：

```
f̂(x) = Σᵢ wᵢφᵢ(x) = Σ_{i∈active(x)} wᵢ
```

**符号说明**：
- m：tiling 数量（套数），决定有效分辨率与泛化程度的权衡
- active(x)：状态 x 激活的瓦片下标集合（每套 tiling 恰好激活一个瓦片，共 m 个激活特征）
- wᵢ：每个瓦片对应的权重

**变量解释**：由于每个状态只激活 m 个瓦片（m 远小于瓦片总数），特征向量高度稀疏，f̂(x) 的计算退化为 m 个权重直接相加。

---
# 4. Basis Function（基函数）

**为什么这样设计**：指示函数（0/1）计算最快，无需浮点运算；多套偏移网格叠加是为了用简单粗糙的基函数集合模拟出精细的连续插值效果（类似摩尔纹叠加产生更精细的图案）。

**Basis长什么样**：每个 φᵢ 是一个矩形（或高维超立方体）区域上的指示函数，区域外恒为0，区域内恒为1。

**举例**：二维状态空间，m=4 套 tiling，每套 5×5 网格，但相对彼此偏移 1/5 个格子——4套叠加后等效分辨率远高于单套 5×5。

**画图（文字描述）**：想象4张透明的方格纸叠在一起，每张方格纸的格子稍微错开一点，任意一点在4张纸上分别落在哪个格子里的"组合编码"几乎是唯一的，这就是有效的高分辨率来源。

---
# 5. Approximation Formula（逼近公式）

统一写成：

```
f(x) = Σᵢ wᵢφᵢ(x)，φᵢ 为分段常数指示函数
```

**如何得到**：预先设计 m 套偏移网格的划分方式（tiling scheme），运行时对每个输入 x 计算落入哪些瓦片（哈希查表实现，常用 tile coding 软件如 Sutton 提供的 tiles3）。

**为什么是线性的**：同样，wᵢ 线性加权求和；且因为每次只有 m 个特征非零，更新规则退化为对这 m 个权重做简单的加法运算，是所有线性基展开方法中计算效率最高的一种。

---
# 6. Mathematical Foundation（数学基础）

- **Approximation Theory**：分段常数逼近（piecewise constant approximation），是最简单的一类函数逼近
- **粗编码理论（Coarse Coding）**：Hinton 提出的用重叠感受野表示连续量的一般框架，Tile Coding 是其规则化、可计算的特例
- **哈希与随机投影**：大规模状态空间中常用哈希函数将瓦片索引压缩到固定大小的表，涉及哈希碰撞的概率分析

---
# 7. Approximation Ability（逼近能力）

**为什么能逼近任意函数**：分段常数函数的集合在瓦片数趋于无穷（分辨率趋于无穷细）时，可以以任意精度逼近任意有界函数（黎曼和的思想）。

**误差如何分析**：误差主要由瓦片粒度（网格宽度）决定，为 O(网格宽度)，即一阶（线性）收敛，明显慢于 Fourier 的谱收敛。

**是否谱收敛**：否，分段常数逼近的收敛阶数很低。

**是否局部收敛**：是，且是所有方法中局部性最强的——每个瓦片的更新完全不影响其他不激活瓦片的权重。

**是否一致收敛**：在网格宽度趋于0时逐点一致收敛到有界函数，但收敛速度慢。

---
# 8. Optimization（参数学习）

- **Least Squares**：理论上可行，但实践中很少用（Tile Coding 通常用于在线学习场景）
- **Gradient Descent / SGD**：最常用，由于特征稀疏，梯度更新只需修改 m 个激活权重
- **TD Learning**：Tile Coding 与 TD(λ) 结合是经典 RL 教材（Sutton & Barto）中的标准范例，更新规则为 wᵢ ← wᵢ + αδ（对激活瓦片）
- **LSTD**：可用，但由于瓦片总数可能很大（尤其哈希表设置很大时），矩阵求逆代价高，实践中较少用 LSTD 配合 Tile Coding

---
# 9. RL中的应用

- **表示V**：V̂(s;w) = Σ_{i∈active(s)} wᵢ，Sutton & Barto 书中 Mountain Car 的标准实现
- **表示Q**：为每个离散动作维护一套独立的瓦片权重，Q̂(s,a) = Σ_{i∈active(s)} wₐ,ᵢ
- **表示Policy**：较少直接用，多用于价值函数一侧
- **对应算法**：Sarsa(λ) with Tile Coding 是 RL 教学中最经典的组合，也是早期 TD-Gammon 之外最成功的函数逼近RL案例之一

---
# 10. Theoretical Properties（理论性质）

- **是否线性**：是（关于参数）
- **是否凸优化**：是
- **是否保证收敛**：线性 TD 收敛理论适用；由于特征稀疏且激活模式简单，实践中收敛表现通常优于稠密基函数
- **是否稳定**：数值上非常稳定（0/1特征，无病态矩阵问题）
- **复杂度**：每次更新/预测 O(m)（m 为 tiling 套数，通常是个位数到几十），与瓦片总数无关——这是最大的工程优势

---
# 11. Advantages（优点）

- 计算极快，适合实时/在线更新场景
- 局部更新，不影响全局，训练稳定
- 实现简单（配合哈希可处理任意大规模离散化）

---
# 12. Disadvantages（缺点）

- 分段常数逼近精度有限，收敛阶数低
- 需要手动设计 tiling 方案（瓦片形状、偏移量、套数），对超参数较敏感
- 高维状态空间下网格数量指数增长（虽然哈希可以缓解内存问题，但表达能力仍受限）

---
# 13. Comparison（与其他方法比较）

| 方法 | 计算速度 | 精度（同参数量下） | 局部性 | 调参难度 |
|---|---|---|---|---|
| Tile Coding | 最快（O(m)） | 较低（分段常数级） | 最好 | 中（需设计tiling） |
| RBF | 中 | 较高 | 好 | 高（中心/带宽） |
| Fourier | 中 | 高（光滑函数） | 差 | 低 |
| CMAC | 快（Tile Coding的前身，类似） | 较低 | 好 | 中 |

---
# 14. Applications（实际应用）

- **Mountain Car**、**Puddle World**：Sutton & Barto 经典教材的标准实验环境
- 早期机器人实时控制系统（对推理速度要求极高的场景）
- 大规模离散化状态空间的 on-policy 控制任务

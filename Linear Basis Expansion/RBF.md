# Radial Basis Function（径向基函数，RBF）
---
# 1. Motivation（为什么提出）

**历史背景**：RBF 方法起源于1970年代的多变量插值问题（Hardy, 1971，用于地形数据插值），1988年 Broomhead & Lowe 将其引入神经网络领域（RBF Network），成为连接经典插值理论与神经网络的重要桥梁。

**解决什么问题**：需要一种**基于"距离"而非"坐标方向"** 的基函数——多项式/Fourier 基函数的值依赖于输入在各坐标轴上的具体位置和方向，而很多物理/控制问题中，函数值只依赖于离某个参考点的**距离**（各向同性），RBF 天然契合这类结构。

**为什么以前的方法不好**：Polynomial/Fourier 的基函数是"轴对齐"的（沿坐标轴振荡），当函数的重要结构是围绕某些"中心点"呈放射状分布时（如价值函数在目标点附近呈同心圆状衰减），轴对齐基函数需要大量项的组合才能模拟这种各向同性结构，效率低。

---
# 2. Mathematical Idea（数学思想）

**一句话解释**：在若干个"中心点"周围放置钟形核函数，输入离中心越近，对应基函数值越大，逼近结果是这些局部钟形响应的加权叠加。

**几何解释**：每个中心 cᵢ 像一个"信号源"，其影响力随距离呈高斯衰减；多个信号源叠加形成一个平滑的地形图，用来拟合目标函数曲面。

**数学解释**：RBF 基函数只依赖于 ||x−cᵢ||（欧氏距离），因此函数值在以 cᵢ 为球心的同心球面上保持不变——这是"径向"（radial）一词的数学含义。

---
# 3. Mathematical Definition（数学定义）

```
φᵢ(x) = exp(−||x−cᵢ||² / (2σᵢ²))    （高斯核，最常用形式）

f̂(x) = Σᵢ₌₁ⁿ wᵢφᵢ(x)
```

**符号说明**：
- cᵢ：第 i 个基函数的中心（需要预先设定或学习得到）
- σᵢ：第 i 个基函数的带宽（控制影响范围的"胖瘦"）
- ||x−cᵢ||：x 到中心 cᵢ 的欧氏距离
- wᵢ：待学习的线性权重

**变量解释**：n（中心数量）、cᵢ（中心位置）、σᵢ（带宽）通常在训练前通过聚类（如 K-means）或均匀网格确定，只有 wᵢ 通过线性回归/RL更新学习——这也是 RBF 相比 MLP 训练更简单、但灵活性较低的原因。

---
# 4. Basis Function（基函数）

**为什么这样设计**：高斯函数具有无限光滑（C∞）、随距离单调衰减、各向同性的性质，是描述"局部影响力"最自然的数学形式；也可选择其他径向核（如多二次曲面核、逆多二次核、薄板样条核）。

**Basis长什么样**：φᵢ(x) 是以 cᵢ 为峰值中心的钟形曲面，随 ||x−cᵢ|| 增大单调衰减到0。

**举例**：一维情形，中心在 x=0.5，σ=0.1，则 φ(x) 在 x=0.5 处取值1，在 x=0.3 或 x=0.7 附近迅速衰减接近0。

**画图（文字描述）**：单个高斯基函数是一座孤立的"小山峰"；多个不同中心的高斯基函数叠加，就像多座山峰组成的地形，峰的高度由权重 wᵢ 决定，峰的位置和"胖瘦"由 cᵢ, σᵢ 决定。

---
# 5. Approximation Formula（逼近公式）

统一写成：

```
f(x) = Σᵢ wᵢφᵢ(x)，φᵢ(x) = exp(−||x−cᵢ||²/2σᵢ²)
```

**如何得到**：先确定中心和带宽（聚类或网格化），再用最小二乘/梯度下降求解线性权重 w。

**为什么是线性的**：与其他基展开方法相同——一旦中心 cᵢ、带宽 σᵢ 固定，f̂ 关于 w 是线性的；但若中心和带宽也作为可学习参数（如 RBF Network 联合训练），则整体模型变为非线性优化问题，这时其性质更接近浅层神经网络。

---
# 6. Mathematical Foundation（数学基础）

- **Approximation Theory**：Micchelli (1986) 证明了在温和条件下，径向基函数插值矩阵可逆，插值问题总有唯一解
- **Kernel Theory**：高斯 RBF 是 Mercer 核的经典例子，对应无穷维特征空间的核方法（与 SVM 中的 RBF 核同源）
- **散乱数据插值理论（Scattered Data Interpolation）**：RBF 最初就是为不规则采样点插值设计的
- **Functional Analysis**：高斯核对应的 RKHS 是 Sobolev 空间的一种特殊情形

---
# 7. Approximation Ability（逼近能力）

**为什么能逼近任意函数**：Park & Sandberg (1991) 证明高斯 RBF 网络具有通用逼近性（Universal Approximation），只要中心数量足够多、位置合理，可以逼近任意连续函数到任意精度。

**误差如何分析**：误差主要由中心密度（覆盖程度）和带宽选择决定；带宽过大导致过度平滑（欠拟合），过小导致基函数几乎不重叠（插值退化，泛化差）。

**是否谱收敛**：不是标准意义上的谱收敛，但对光滑函数，误差随中心数量增加以多项式速率下降。

**是否局部收敛**：是——每个基函数只在中心附近有显著响应，具有较好的局部性（局部性强弱由带宽σ控制）。

**是否一致收敛**：中心覆盖足够密、带宽选择合理时可以做到一致收敛；带宽选择不当会导致某些区域逼近质量差。

---
# 8. Optimization（参数学习）

- **Least Squares**：中心/带宽固定后，权重 w 的求解是标准线性最小二乘问题，有闭式解
- **Gradient Descent**：若联合优化中心、带宽、权重（RBF Network），则需要梯度下降处理非凸问题
- **TD Learning**：RL 中权重 w 通过半梯度 TD 更新，中心/带宽通常预先固定（避免非平稳目标导致训练不稳定）
- **LSTD**：适用于权重求解，尤其在中心数量适中（几十到几百）的问题中

---
# 9. RL中的应用

- **表示V**：V̂(s;w) = Σᵢ wᵢexp(−||s−cᵢ||²/2σᵢ²)，经典应用于连续状态空间的价值函数逼近
- **表示Q**：Q̂(s,a;w) = Σᵢ wᵢφᵢ(s,a)，中心同时覆盖状态-动作联合空间
- **表示Policy**：可用于连续动作策略的均值函数参数化（如高斯策略的均值由 RBF 网络输出）
- **对应算法**：早期连续控制RL（如倒立摆、机械臂）中广泛使用 RBF + Sarsa/Actor-Critic 的组合

---
# 10. Theoretical Properties（理论性质）

- **是否线性**：中心/带宽固定时关于权重线性；联合训练时非线性
- **是否凸优化**：固定中心情形下是凸问题；联合优化中心时非凸
- **是否保证收敛**：固定中心时满足线性TD收敛理论；联合训练场景收敛性理论较弱
- **是否稳定**：带宽选择不当（过窄）会导致设计矩阵病态，数值不稳定
- **复杂度**：预测/更新复杂度为 O(n)（n为中心总数，若不做稀疏化则不像 Tile Coding 那样只激活少数基函数）

---
# 11. Advantages（优点）

- 各向同性、局部性好，适合描述"围绕关键点衰减"的价值函数结构
- 数学理论完备（插值理论、通用逼近性均有严格证明）
- 与核方法/高斯过程理论紧密相连，便于扩展到贝叶斯设定

---
# 12. Disadvantages（缺点）

- 中心和带宽的选择对性能影响巨大，需要调参或额外的聚类步骤
- 高维空间中中心数量同样面临维度灾难（需要的中心数随维度指数增长以保持覆盖密度）
- 不像 Tile Coding 那样天然稀疏，标准 RBF 每次预测需计算所有基函数（除非做局部截断）

---
# 13. Comparison（与其他方法比较）

| 方法 | 局部性 | 是否需调中心/带宽 | 高维扩展性 | 与核方法关系 |
|---|---|---|---|---|
| RBF | 好 | 是（关键超参数） | 中等（维度灾难） | 直接对应（Mercer核） |
| Tile Coding | 好 | 需设计tiling方案 | 好（哈希缓解） | 无直接对应 |
| Fourier | 差 | 否（仅调阶数） | 差 | 无直接对应 |
| Wavelet | 好 | 需选母小波 | 中等 | 部分相关 |

---
# 14. Applications（实际应用）

- **连续控制任务**：倒立摆平衡、机械臂轨迹跟踪等经典连续状态-动作空间控制
- **Gaussian Process RL**：将 RBF 的确定性版本推广为带不确定性估计的贝叶斯方法
- 早期 Actor-Critic 方法中连续动作策略的函数逼近器
